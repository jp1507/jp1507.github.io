## 0. 개요

---

Github 사용자라면 누구나 레포지토리를 생성할 때 `README.md`라는 파일을 한 번쯤 본 적이 있을 것이다. 확장자가 `.md`로 된 파일이 바로 마크다운 파일이고 쉬운 문법으로 다양한 플랫폼을 지원한다. 이 글은 블로그에 사용하는 마크다운 문법에 대한 총 정리 글이다.

### 장단점

1. 장점
   - 쉬운 문법
   - 간편한 관리
   - 지원되는 다양한 플랫폼 및 프로그램  
  
2. 단점
   - 표준이 없어 사용자마다 문법이 조금씩 다름
   - 모든 HTML 마크업을 대체하지 못함

### 사용처

메모장부터 전용 에디터까지 많은 곳에서 사용할 수 있으며, VS Code같은 에디터에서 편집하는 것을 추천한다. 에디터는 마크다운 문법으로 작성된 문서의 미리보기를 실시간으로 확인할 수 있는 기능이 있으므로 편리하다.

## 1. 강조

---
입력:

```markdown
1. 이텔릭체는 *이텔릭체* 또는 _이텔릭체_ 로 표현
2. 볼드체는 **볼드체** 또는 __볼드체__ 로 표현 (__*이텔릭체*와 볼드체__ 로 혼합표현)
3. 취소선은 ~~취소선~~ 로 표현
4. <u>밑줄</u>은 마크다운 지원 X, HTML의 `<u></u>`태그 사용
```

출력:

1. 이텔릭체는 *이텔릭체* 또는 _이텔릭체_ 로 표현
2. 볼드체는 **볼드체** 또는 __볼드체__ 로 표현 (__*이텔릭체*와 볼드체__ 로 혼합표현)
3. 취소선은 ~~취소선~~ 로 표현
4. <u>밑줄</u>은 마크다운 지원 X, HTML의 `<u></u>`태그 사용  

## 2. 제목

---
<!-- markdownlint-disable -->
> 하나의 `# 제목`만 사용 가능하고 글 제목에서 사용 중이므로  `## 제목`부터 사용.
{: .prompt-warning }
<!-- markdownlint-restore -->

```markdown
# 제목1
## 제목2
### 제목3
#### 제목4
##### 제목5
###### 제목6
```
`# 제목`, `## 제목2`는 다음과 같이 표현 가능하다.  
```markdown
제목1
======
제목2
------
```

## 3. 목록

---

### 순서 목록

입력:

```markdown
1. 항목
2. 항목
   1. 항목
   2. 항목
3. 항목
```

출력:

1. 항목
2. 항목
   1. 항목
   2. 항목
3. 항목  

### 비순서 목록

입력:  

```markdown
- 항목
- 항목
   - 항목
   - 항목
- 항목
```
출력:  

- 항목
- 항목
  - 항목
  - 항목
- 항목

### 할일 목록

입력:  
```markdown
- [ ] 할일
   - [X] 목표1
   - [ ] 목표2
```
출력:  

- [ ] 할일
  - [X] 목표1
  - [ ] 목표2

### 설명 목록

입력:

```markdown

```
출력:  

설명 목록
: 세미 콜론을 이용해서 사용한다
: 세미 콜론을 이용해서 사용한다

## 6. 표

---
표의 제목행을 구분하기 위해 3개 이상의 `-`기호를 사용한다. 이때 `:`기호를 이용해 데이터행의 정렬에 대해 정의할 수 있다.
```markdown
| 헤더 | 헤더 | 헤더  |
| ---- | ---: | :---: |
| 셀   |   셀 |  셀   |
| 셀   |   셀 |  셀   |
```
입력:  

```markdown
| Company | Domain     |           Country |
| :-----: | :--------- | ----------------: |
| Google  | google.com |               USA |
|  Naver  | naver.com  | Republic of Korea |
| Github  | github.com |               USA |
```
출력:  

| Company |     Domain |           Country |
| :-----: | ---------: | ----------------: |
| Google  | google.com |               USA |
|  Naver  |  naver.com | Republic of Korea |
| Github  | github.com |               USA |

> `|` 기호를 출력하고 싶다면 이스케이프 처리해서 사용하면 된다.
{: .prompt-tip }

### 수식

수식은 [**MathJax**](https://www.mathjax.org/)를 이용하면 된다.

입력:  

```text
$$
\begin{equation}
  \sum_{n=1}^\infty 1/n^2 = \frac{\pi^2}{6}
  \label{eq:series}
\end{equation}
$$

방정식을 \eqref{eq:series}로 참조할 수 있다.

$a \ne 0$일 때, $ax^2 + bx + c = 0$의 해는 2개이고 다음과 같다.
$$ x = {-b \pm \sqrt{b^2-4ac} \over 2a} $$
```
출력:  

$$
\begin{equation}
  \sum_{n=1}^\infty 1/n^2 = \frac{\pi^2}{6}
  \label{eq:series}
\end{equation}
$$

방정식을 \eqref{eq:series}로 참조할 수 있다.

$a \ne 0$일 때, $ax^2 + bx + c = 0$의 해는 2개이고 다음과 같다.  
$$ x = {-b \pm \sqrt{b^2-4ac} \over 2a} $$

## 9. 하이퍼 링크

---
일반 URL과 `<>`로 수식된 URL은 자동으로 링크로 사용한다.
```markdown
[사이트이름](하이퍼링크)
[사이트이름](하이퍼링크 "설명")
[사이트이름][참조]

[참조]: 하이퍼링크
[참조]: 하이퍼링크 "설명"
```
입력:  

```markdown
[Google](https://google.com "https://google.com")  
[Google][참조 링크1]  
[내 블로그][참조 링크2]  
[글 목록 보기](../posts)  

문서 내에 [참조 링크1]를 재사용 가능  

[참조 링크1]: https://google.com  
[참조 링크2]: https://jp1507.github.io  
```
출력:  

[Google](https://google.com "https://google.com")  
[Google][참조 링크1]  
[내 블로그][참조 링크2]  
[글 목록 보기](../posts)  

문서 내에 [참조 링크1]를 재사용 가능  

[참조 링크1]: https://google.com  
[참조 링크2]: https://jp1507.github.io
