## Chapter 11. 상속 : 자식 클래스(Derived Class)들의 생성 순서

### 부모로부터 상속 받은 것은 온전히 자기 자신의 것

---

- 부모로부터 상속 받은 멤버들은 자기 자신의 멤버들이 되기 때문에 `this` 포인터로 참조하여도 무방

```cpp
class Mother
{
public:
    int m_i;    
};
class Child : public Mother
{
public:
    Child()
    {
        this->m_i = 10;
        this->Mother::m_i = 1024;
    }
}
```

### 부모 생성자는 자식 생성자보다 무조건 먼저 호출됨

---

- 가장 부모, 조상인 클래스의 생성자부터 호출됨

```cpp
class Mother
{
public:
    int m_i; 

    Mother() : m_i(0)
    {
        cout << "Mother construction" << endl; 
    }   
};
class Child : public Mother
{
public:
    double m_d;

    Child() : m_d(0.0)
    {
        cout << "Child construction" << endl; 
    }
};

int main()
{
    Child c;
    return 0;
}
```

- `Child c`
  - 자식 생성자에서 부모의 생성자 호출을 직접 명시하지 않으면 부모의 디폴트 생성자를 호출함
    - 부모 클래스에 생성자가 정의되었지만 디폴트 생성자가 없다면 에러 발생
  - 부모의 생성자 호출을 명시하면 인수에 맞게 오버로딩된 생성자를 호출함

### 초기화 리스트에서의 순서

---

- 부모 생성자를 먼저 호출하여 부모 멤버들의 초기화부터 실행

```cpp
Child() : m_d(1.0), Mother(1024) {}
```

- 부모 생성자를 자식 생성자의 초기화 리스트에서 직접 호출할 때
  - 메모리는 자식이 먼저 할당받을 수 있지만 초기화에서만큼은 무조건 부모 멤버들이 먼저 초기화됨

```cpp
Child() : m_d(1.0) {}
```

- 부모 생성자를 자식 생성자의 초기화 리스트에서 호출하지 않을 때
  - 쓰레기 값만 들어있는 상태에서 부모 디폴트 생성자를 먼저 실행
