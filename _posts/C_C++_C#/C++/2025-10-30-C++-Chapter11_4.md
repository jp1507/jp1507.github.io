## Chapter 11. 상속 : 상속과 접근 지정자

### public 상속

---

- `public`으로 상속받게 되면 원래의 접근 지정자 성격이 유지됨

```cpp
class Mother
{
public:
    int m_public;
protected:
    int m_protected;
private:
    int m_private;
};
class Daughter : public Mother
{
public:
    Daughter()
    {
        m_public = 7;
        m_protected = 7;
        m_private = 7;    // Error !!!
    }
}

int main()
{
    Daughter daughter;

    daughter.m_public = 100;
    daughter.m_protected = 100;   // Error !!!
    daughter.m_private = 100;   // Error !!!
}

```

### protected 상속

---

- `protected`로 상속을 받게되면 더 엄격해져, 모든 자식, 손자 클래스들 안에서 멤버를 사용할 수 있고 외부에서만 사용할 수 없으며 `private` 멤버는 모든 자식, 손자 클래스들 내에서도 접근 안됨

```cpp
class Mother
{
public:
    int m_public;
protected:
    int m_protected;
private:
    int m_private;
};
class Daughter : protected Mother
{
public:
    Daughter()
    {
        m_public = 7;
        m_protected = 7;
        m_private = 7;    // Error !!!
    }
}

int main()
{
    Daughter daughter;

    daughter.m_public = 100;    // Error !!!
    daughter.m_protected = 100;    // Error !!!
    daughter.m_private = 100;    // Error !!!
}
```

### private 상속

---

- `private`으로 상속받으면 자식 클래스의 멤버가 모두 `private`화되어 손자, 즉 한번 더 상속이 일어나면 모든 멤버에 접근하지 못하고 직속 자식 클래스에서만 접근 가능함

```cpp
class Mother
{
public:
    int m_public;
protected:
    int m_protected;
private:
    int m_private;
};
class Daughter : private Mother
{
public:
    Daughter()
    {
        m_public = 7;
        m_protected = 7;
        m_private = 7;    // Error !!!
    }
}
class GrandDaughter : public Daughter
{
public:
    GrandDaughter()
    {
        m_public = 24;    // Error !!!
    }
}

int main()
{
    Daughter daughter;

    daughter.m_public = 100;    // Error !!!
    daughter.m_protected = 100;    // Error !!!
    daughter.m_private = 100;    // Error !!!
}
```
