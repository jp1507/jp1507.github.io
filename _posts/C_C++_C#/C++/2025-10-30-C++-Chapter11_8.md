## Chapter 11. 상속 : 다중 상속

### 다중 상속

---

- 한 자식 클래스에 여러 부모 클래스를 상속시키는 것

```cpp
class USBDevice
{
private:
    long m_id;
public:
    USBDevice(long id) : m_id(id) {}

    long GetID() { return m_id; }
    void PlugAndPlay() {}
};
```
{: file="USBDevice.h"}

```cpp
class NetworkDevice
{
private:
    long m_id;
public:
    class NetworkDevice(long id) : m_id(id) {}

    long GetID() { return m_id; }
    void Networking() {}
};
```
{: file="NetworkDevice.h"}

```cpp
class USBNetworkDevice : public USBDevice, public NetworkDevice
{
public:
    USBNetworkDevice(long usb_id, long net_id) :USBDevice(usb_id), NetworkDevice(net_id) {}
};
```
{: file="USBNetworkDevice.h"}

- `USBNetworkDevice` 클래스는 `USBDevice`와 `NetworkDevice` 2개의 부모 클래스를 상속받음
  - `mydevice.Networking();` : `NetworkDevice`로부터 상속
  - `mydevice.PlugAndPlay();` : `USBDevice`로부터 상속
- 언어에 따라 다중 상속을 지원하지 않는 언어도 존재

### 다중 상속시 발생할 수 있는 문제

---

- 다중 상속은 다음과 같은 문제들을 일으킬 수 있기 때문에 신중히 사용하는 것이 좋음

1. 두 부모에게 중복되는 멤버가 있을 때문에

    ```cpp
    long GetID() { return m_id; }
    ```

    - 두 부모 클래스에 `GetID()` 함수가 중복되어 있어 자식 클래스에서 이 함수 호출 시 모호해 에러가 발생함

    ```cpp
    int main()
    {
        USBNetworkDevice my_device(3.14, 6.022);

        my_device.USBDevice::getID();
        my_device.NetWorkDevice::getID();
    }
    ```

    - 부모 클래스 이름을 앞에 명시해 어떤 부모의 함수인지 명확히 하는 것으로 해결할 수 있음  

2. 다이아몬드 상속

    ![이미지](https://imgur.com/0OxbXOr.png)

    - `C` 클래스는 `A` 클래스를 상속받는 `B1`, `B2`를 다중 상속받기 때문에 `C` 클래스에 `A`로부터 상속받은 부분이 두 번 중복 상속되는 문제가 발생함
    - 가상 상속 받기의 방법으로 해결할 수 있음
